## 🔄 SYNC — 세션 시작 프로토콜

유저가 **"sync"** 또는 **"싱크"** 라고 말하면 (혹은 새 세션 시작 시):

1. **아래 순서로 읽기**:
   - `CLAUDE.md` (이 파일) — Claude 전용 지침
   - `ROBOT.md` — 공유 프로젝트 규칙, 배포 설정, 워크플로우
   - `TODO.md` — 현재 작업 및 버전 파이프라인
   - `README.md` — 기능/로드맵 파악용 (운영 규칙 아님)
2. **최근 변경사항 확인**: `git log --oneline -10`
3. **유저에게 보고**:
   - 현재 브랜치 및 최신 커밋
   - 지난 세션 이후 변경사항 (문서, 기능, 버그 수정)
   - 열린 PR 또는 대기 중인 배포 작업
   - 다음 작업 (TODO.md의 첫 번째 `🔲` 항목)
4. **문서 간 불일치 사항 발견 시 즉시 플래그**

> 공유 규칙 (Safety Harness, 배포 규율, 릴리즈 워크플로우)은 `ROBOT.md`에 있음 — 모든 공유 운영 규칙의 단일 진실 소스.

## 📊 Context 관리 규칙

**Context 70% 이상일 때**:
- 자동으로 `/compact` 실행하여 대화 히스토리 압축
- 유저에게 사전 공지 후 진행

**Context 100% 접근 시**:
- 항상 경고 메시지 표시
- 사용자 확인 후 `/compact` 실행
- 압축 후 정상적으로 작업 재개

이유: 장시간 세션에서 컨텍스트 손실 방지 및 세션 연속성 보장

### 로컬 개발

```bash
cd /home/ubuntu/projects/Junmo-game
npm start        # 개발 서버 (localhost:3000)
npm run build    # 프로덕션 빌드
```

> `.env.local` 파일에 API 키 보관 (절대 커밋 금지):
> ```
> REACT_APP_ANTHROPIC_KEY=...
> REACT_APP_GROQ_KEY=...
> ```
